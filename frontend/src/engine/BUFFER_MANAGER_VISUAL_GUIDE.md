## Common Patterns

### Pattern 1: Single Frame Processing
```typescript
const buffer = bufferManager.getBuffer(width, height);
try {
  // Process frame
  const result = await processFrame(buffer);
  return result;
} finally {
  bufferManager.returnBuffer(buffer);
}
```

### Pattern 2: Continuous Processing
```typescript
function processLoop() {
  const buffer = bufferManager.getBuffer(width, height);
  
  // Process frame
  processFrame(buffer);
  
  // Return and continue
  bufferManager.returnBuffer(buffer);
  requestAnimationFrame(processLoop);
}
```

### Pattern 3: Multi-Buffer Processing
```typescript
const inputBuffer = bufferManager.getBuffer(width, height);
const outputBuffer = bufferManager.getBuffer(width, height);

try {
  // Process using both buffers
  await processWithBuffers(inputBuffer, outputBuffer);
} finally {
  bufferManager.returnBuffer(inputBuffer);
  bufferManager.returnBuffer(outputBuffer);
}
```

## Troubleshooting Guide

```
Problem: High memory usage
Solution: 
  1. Check buffer count: getBufferCount()
  2. Check memory: getMemoryUsageMB()
  3. Force cleanup: forceCleanup()
  4. Reduce resolution if needed

Problem: Buffers not being reused
Solution:
  1. Verify returnBuffer() is called
  2. Check buffer dimensions match
  3. Review buffer lifecycle

Problem: Performance degradation
Solution:
  1. Monitor GC pauses
  2. Check memory health
  3. Verify buffer pooling is working
  4. Profile allocation patterns
```
